# 版本发布异常汇总

## 简答
> 记录历史版本发布过程中遇到的各类异常问题，包括渠道拒审、DLC问题、兼容问题等，以及对应的处理方案和后续规避措施。

---

## 常见拒审原因及处理

### OPPO 渠道常见拒审原因

| 拒审原因 | 处理方式 |
|----------|----------|
| 公告栏内有跳转链接/导流内容 | 运营下掉国服公告 |
| 敏感词未屏蔽 | 热更 1780 表 S_MAP_ban_words 字段添加屏蔽词 |
| 概率活动未展示概率公式 | 在测试说明中备注概率展示入口位置 |
| 跳转浏览器功能 | 客户端屏蔽相关入口，重新打包 |
| 非 OPPO 广告插件 | 屏蔽游戏内广告，重新打包 |

### 小米渠道常见拒审原因

| 拒审原因 | 处理方式 |
|----------|----------|
| 游戏内含有导流内容（公告/邮件） | 运营删除相关公告或邮件 |
| 隐私政策不合规 | 修改隐私协议内容后重新出包 |
| 游戏界面显示英文 | TGS auditing 配置屏蔽相关功能 |
| 敏感词屏蔽未生效 | 服务器修改代码后部署，热更屏蔽词表 |

### 华为渠道常见拒审原因

| 拒审原因 | 处理方式 |
|----------|----------|
| 健康游戏忠告文字不清晰 | 修改 loading 界面底部小字（放大+描边） |
| 检测到第三方广告插件 | 去掉广告 SDK，重新打包 |
| 账号注销服务缺失 | 发 patch 添加注销功能 |
| 版署实名认证未接入 | 完成版署网络游戏防沉迷实名认证系统接入 |
| 登录返回后无法再次登录 | 客户端处理后重新出包 |

### 九游渠道常见拒审原因

| 拒审原因 | 处理方式 |
|----------|----------|
| 酒吧抽奖违规 | 申诉或修改相关内容 |
| 射击小游戏溅血内容 | 屏蔽射击小游戏选择，重新出包 |
| 需要最新 SDK 版本 | 平台接入最新 SDK 后重新出包 |

### iOS 国服常见拒审原因

| 拒审原因 | 处理方式 |
|----------|----------|
| 无 ATT 弹窗（应用请求跟踪） | 客户端加上弹窗，重新出包 |
| 兑换码入口未屏蔽 | 登录审核账号切到审核服 |
| DLC 下载时间过长 | 与超哥确认是否需要处理 |
| 一键登录无法使用（iPad） | 说明一键登录需要手机卡 |

### iOS 越南包常见拒审原因

| 拒审原因 | 处理方式 |
|----------|----------|
| 无 ATT 弹窗 | 客户端添加，重新出包 |
| 系统权限弹窗非越南语 | 客户端修改本地化，重新出包 |
| 强制更新无下载大小提示 | TGS 后台加字段限制 |
| Age rating 与游戏内不符 | 修改商店信息 |

## 线上屏蔽词添加方式

### 方式一：TGS 后台（聊天频道）
- **路径**：TGS 后台 → 聊天 → 关键字管理
- **生效时间**：聊天服会在 1 小时刷新周期后自动生效

### 方式二：热更配置表（非聊天场景）
- **配置表**：1780 表 S_MAP_ban_words 字段
- **生效时间**：热更后线上服务器即时生效

## DLC 发布常见问题

### 兼容 DLC 问题

| 问题类型 | 原因 | 规避措施 |
|----------|------|----------|
| 兼容分支缺少内容 | 兼容分支拉取太早 | 出正式兼容 DLC 前，从 hotfix 往兼容分支再合一次 |
| 配置/资源丢失 | 中间版本合入主分支时遗漏 | 策划负责人比对修改过的配置内容 |
| 兼容处理不恰当 | 修改方案未充分评估 | 兼容 bug 标题写清楚修改前后表现和影响范围 |

### Patch 与 Hotfix 问题

| 问题类型 | 原因 | 规避措施 |
|----------|------|----------|
| 拉到 patch 后游戏异常 | patch 和 hotfix 存在依赖关系 | 降低发 patch 频率，验证裸包+已有 hotfix 场景 |
| 高版本覆盖低版本 | 不同分支的资源版本号冲突 | 确保分支一致，高版本包含低版本内容 |
| Code Patch Failed | patch 函数签名与老版本不兼容 | 回退 DLC，处理后重新发布 |

### Ondemand 问题

| 问题类型 | 原因 | 规避措施 |
|----------|------|----------|
| 更新后小游戏挂掉 | 只更新 ondemand 未拉到 hotfix | 验证只更新 ondemand 后小游戏流程是否正常 |
| 资源需要重新下载 | 资源从包内移到 ondemand | 与 PD 确认需求预期效果 |
| 包体过大 | 不需进包的资源被打进包 | 检查 ondemand 配置 |

## 版本发布流程规范

### 正式 DLC 验证前

1. 版本负责人根据当前版本功能情况，选择对应线上服
2. 请服务器版本负责人 down 到 beta 环境，更新到当前版本功能分支
3. 各自顶该服线上号进入验证功能
4. 选择账号后在测试群内同步，避免顶到同一个号

### 配置相关验证

1. 确认是双端均需更新，还是仅单端更新
2. 若双端更新，确认更新顺序（先热更配置 or 先发 DLC）
3. 对应验证老包新服、新包老服场景
4. 验证通过后及时同步给策划/运营，提醒热更线上

### 合服部署注意事项

- 合服处理和测试统一使用与线上同步的分支
- 根据更新时间和内容判断用 hotfix 还是单独线上分支
- 合服需在版本部署前操作

## 顶号方式

### 方式一：GM 指令（测试环境）
```
charprox 账号的角色id
```
然后重启客户端

### 方式二：LUU 指令
```
luu 账号的udid
```
在运营后台查询 UDID：信息查询 → 玩家查询 → 输入角色 ID

### 方式三：Unity 账号工具
1. 关闭游戏
2. 点击 Framework → Account → Account Tools
3. 在本地 UDID 栏输入要顶号玩家的 UDID
4. 选择环境（gold/beta）
5. 重新进入游戏

## 常见坑点

- **发布时间**：避免晚上 10 点后发布 DLC 或部署全服，确保有问题能及时处理
- **分支合并**：有版本功能发布时，QA 测试完成后需提前通知三方进行合并分支
- **运营需求**：如有特殊上线更新方式，测试和实际上线必须流程同步
- **Unity 测试**：用 Unity 测试的功能，最后也需要用真机过一遍
- **审核服**：iOS 国服/越南/国服安卓渠道在验包和未过审阶段会配置 auditing
