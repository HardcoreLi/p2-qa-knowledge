# 合服测试流程

## 简答
> 服务器合并时的测试验证流程，确保玩家数据正确迁移、功能正常。

## 测试时机
- 合服维护前：准备测试
- 合服维护后：验证测试

## 合服前准备

### 1. 获取合服信息
- 源服务器列表
- 目标服务器
- 合服时间
- 数据处理规则

### 2. 准备测试账号
- 在各源服务器创建测试账号
- 记录账号数据（等级、资源、联盟等）

### 3. 确认测试重点
- 数据迁移正确性
- 冲突处理规则
- 功能兼容性

## 合服后测试

### 1. 账号登录验证
- [ ] 各源服账号能正常登录
- [ ] 登录后进入正确的服务器
- [ ] 账号绑定关系正确

### 2. 数据验证
- [ ] 角色等级正确
- [ ] 资源数量正确
- [ ] 道具数量正确
- [ ] 英雄数据正确
- [ ] 建筑数据正确

### 3. 联盟验证
- [ ] 联盟数据正确
- [ ] 联盟成员正确
- [ ] 联盟排名正确
- [ ] 联盟建筑正确

### 4. 排行榜验证
- [ ] 个人排行榜正确
- [ ] 联盟排行榜正确
- [ ] 历史记录正确

### 5. 冲突处理验证
- [ ] 重名处理正确
- [ ] 坐标冲突处理正确
- [ ] 联盟名冲突处理正确

### 6. 功能验证
- [ ] 跨服功能正常
- [ ] 聊天功能正常
- [ ] 邮件功能正常
- [ ] 好友功能正常

## 测试报告

### 报告模板
```
【合服测试报告】
合服信息：xxx服 + xxx服 → xxx服
测试时间：xxxx-xx-xx
测试人：xxx

一、测试结果：通过/不通过

二、数据验证：
- 账号A：✓ 数据正确
- 账号B：✓ 数据正确

三、功能验证：
- 联盟功能：✓
- 排行榜：✓
- 跨服功能：✓

四、发现问题：
- 问题1：xxx
- 问题2：xxx

五、建议：
xxx
```

## 常见坑点
- **数据丢失**：仔细核对关键数据
- **排名变化**：合服后排名会重新计算
- **联盟冲突**：注意联盟名重复处理
- **坐标迁移**：注意坐标冲突处理规则
