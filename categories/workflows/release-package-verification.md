# 正式包验证流程

## 简答
> 同分支+同打包工程的不同渠道包常规功能无差异，只需完整验证其中一个包，其他渠道包验证差异部分（登录、支付、FTE等）即可。验证完成后上传 APK 至 box。

---

## 验证原则

**核心原则**：同分支 + 同一个打包工程构建出来的不同渠道包，常规功能无差异

**验证策略**：
- 选一个包**完整验证常规功能**
- 其他包只验证**差异部分**（登录、支付、FTE 等）

---

## 各渠道验证方案

### 1. 国际服安卓主包组（GP + 马甲包 + 越南）

| 包名 | 验证范围 |
|------|----------|
| **GP 主包** | ✅ 完整验证常规 |
| mj1 ~ mj6 | 差异验证：FTE、登录、支付 |
| 越南包 | 差异验证：FTE、登录、支付 |

> 💡 一般无特殊情况，这几个包可一起出。若分开出包，需在每次构建出来的包中按此规则验证。

## 差异验证重点

各渠道包需重点验证的差异内容：

| 验证项 | 说明 |
|--------|------|
| **FTE** | 新手引导流程 |
| **登录** | 各渠道登录方式、账号绑定 |
| **支付** | 各渠道支付流程、支付回调 |

---

## 验证后操作

> ⚠️ **特别提醒**：验证完成后上传对应整包 APK 至 box

---

## 验证流程总结

```
确认打包分支和工程
    ↓
选定完整验证的主包
    ↓
主包完整验证常规功能
    ↓
其他渠道包验证差异（登录/支付/FTE）
    ↓
上传 APK 至 box
```

---

## 常见坑点

- **分开出包时遗漏验证**：如果不是一起出包，每次构建都需要按规则验证
- **忘记上传 box**：验证完成后必须上传 APK
- **差异验证不全**：登录、支付、FTE 都需要验证，不能只验证其中一项
- **选错完整验证的包**：注意各组的主包是哪个（GP/iOS主包/Main包）
