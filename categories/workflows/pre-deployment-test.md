# 上线前测试流程

## 简答
> 版本正式上线前的最终验证流程，确保版本质量达到上线标准。

## 测试时机
- 版本提审前
- 版本正式发布前

## 测试环境

### 提审包测试
- 使用提审包（非测试包）
- 连接预发布环境或正式环境
- 模拟真实玩家环境

### 环境要求
- 正式签名包
- 正式服务器配置
- 正式 TGS 配置

## 测试内容

### 1. 核心功能验证
- [ ] 登录/注册正常
- [ ] 核心玩法正常
- [ ] 充值功能正常
- [ ] 数据存储正常

### 2. 新功能验证
- [ ] 本版本新功能完整
- [ ] 新功能无严重 bug
- [ ] 新功能体验正常

### 3. 回归测试
- [ ] 老功能无影响
- [ ] 关键路径正常
- [ ] 无明显性能问题

### 4. 兼容性测试
- [ ] 主流设备测试
- [ ] 不同系统版本测试
- [ ] 不同网络环境测试

### 5. 合规性检查
- [ ] 隐私政策更新
- [ ] 用户协议更新
- [ ] 实名认证正常（国服）
- [ ] 防沉迷正常（国服）

## 上线检查清单

### 客户端
- [ ] 包版本号正确
- [ ] 包签名正确
- [ ] 图标/名称正确
- [ ] 无测试代码残留

### 服务端
- [ ] 服务器配置正确
- [ ] 数据库准备就绪
- [ ] 运维脚本准备就绪

### 配置
- [ ] TGS entrance 配置正确
- [ ] 活动配置正确
- [ ] 公告配置正确
- [ ] 强更配置准备

### 运营
- [ ] 版本公告准备
- [ ] 补偿方案准备（如需）
- [ ] 客服话术准备

## 风险评估

### 高风险项
- 涉及充值的改动
- 涉及数据的改动
- 涉及核心玩法的改动

### 应对措施
- 准备回退方案
- 准备紧急修复方案
- 安排值班人员

## 测试报告

### 报告模板
```
【上线前测试报告】
版本：x.x.x
测试时间：xxxx-xx-xx
测试人：xxx

一、测试结论：可以上线 / 建议延期

二、测试覆盖：
- 核心功能：✓
- 新功能：✓
- 回归测试：✓
- 兼容性：✓

三、遗留问题：
- 问题1：xxx（影响等级：低）
- 问题2：xxx（影响等级：中）

四、风险提示：
xxx

五、建议：
xxx
```

## 常见坑点
- **测试包与提审包不同**：确保测试的是提审包
- **环境差异**：测试环境与正式环境配置差异
- **时区问题**：活动时间配置注意时区
- **缓存问题**：清除缓存后测试
